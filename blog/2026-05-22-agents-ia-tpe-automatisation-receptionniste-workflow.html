Automatisation TPE : comment un workflow n8n + agent vocal gère vos appels sans vous

Pas de magie. Voici les étapes exactes, outil par outil, de ce qui se passe quand un client appelle votre numéro et que vous n'êtes pas disponible.

AA

Abdallah Ait Essaghir

ABEMA AGENCY — Fondateur

Les deux outils du pipeline

L'automatisation ABEMA repose sur deux technologies open-source ou SaaS :

  • n8n — outil d'automatisation de workflows, auto-hébergé sur un VPS Hostinger. C'est le chef d'orchestre : il reçoit les données, déclenche les actions, connecte les outils entre eux [1]
  • Synthflow — plateforme d'agents vocaux IA. Elle gère la couche téléphonie (SIP), la reconnaissance vocale, le LLM et la synthèse vocale [2]

Ces deux outils communiquent via des webhooks (appels HTTP). n8n écoute les événements de Synthflow et déclenche les actions en conséquence dans vos outils métier.

Le flux complet, étape par étape

1
L'appel arrive sur votre numéro

Votre numéro de téléphone professionnel est configuré pour transférer vers Synthflow (via SIP ou numéro Synthflow dédié) quand vous ne décrochez pas après N sonneries — ou en permanence si vous préférez.

2
Synthflow décroche et identifie l'appelant

L'agent joue votre message d'accueil personnalisé, indique qu'il est un assistant IA, et commence la conversation. Il demande le nom de l'appelant, l'objet de l'appel et ses disponibilités si nécessaire.

3
L'agent qualifie la demande

Selon le script paramétré pour votre activité, l'agent pose les questions utiles : type de prestation, localisation, urgence, budget indicatif. Il ne pose que les questions pertinentes — si l'appelant demande simplement vos horaires, il répond directement.

4
Synthflow envoie les données à n8n via webhook

En fin d'appel, Synthflow pousse un objet JSON structuré vers n8n : nom, numéro, objet, données collectées, durée, transcription de l'appel. C'est le signal de départ pour les automatisations.

5
n8n orchestre les actions métier

n8n reçoit le webhook et déclenche en parallèle selon votre configuration :

· Création ou mise à jour d'une fiche contact dans votre CRM (HubSpot, Hektor, Apimo)

· Vérification de vos disponibilités dans Google Calendar

· Si RDV demandé : proposition d'un créneau et confirmation dans le calendrier

· Envoi d'un SMS ou email de confirmation à l'appelant

· Notification vers vous (SMS, email ou Telegram) avec le résumé de l'appel

6
Vous consultez le résumé

À votre retour, vous avez un message avec : qui a appelé, pourquoi, ce que l'agent a dit, et l'action déclenchée (RDV planifié, fiche créée, etc.). Aucune donnée perdue, même si vous étiez indisponible 3 heures.

Les intégrations disponibles

n8n peut se connecter à plusieurs centaines d'outils via ses connecteurs natifs ou via l'API REST générique. Les intégrations testées et déployées par ABEMA :

Google Calendar HubSpot CRM Hektor (artisans) Apimo (immobilier) Gmail / Outlook Telegram (alertes)

Toute intégration avec une API REST peut être connectée — si votre logiciel métier expose une API, n8n peut s'y brancher.

Ce que ce pipeline ne fait pas (encore)

Limites honnêtes

  • · L'agent ne peut pas accéder à des systèmes sans API (logiciel métier avec base de données locale uniquement)
  • · Les appels bruités (chantier, environnement bruyant) peuvent réduire la précision de transcription
  • · Les situations complexes (litige, demande hors script, client mécontent) sont redirigées vers vous — pas traitées automatiquement
  • · La mise en place nécessite un VPS auto-hébergé (Hostinger KVM 2, ~7 €/mois) pour faire tourner n8n — ABEMA le configure pour vous
