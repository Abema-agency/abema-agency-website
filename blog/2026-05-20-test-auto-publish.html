Comment ABEMA publie automatiquement ses articles de blog — pipeline n8n → GitHub → Vercel

Un workflow n8n génère l'article, le pousse directement sur GitHub, et Vercel redéploie en 90 secondes. Sans développeur, sans CMS, sans intervention humaine.

Cet article a été publié automatiquement par un pipeline n8n. Voici comment il fonctionne — et ce que cela signifie pour une TPE qui veut automatiser sa présence en ligne.

Le pipeline en 4 étapes

  1. Déclencheur planifié — n8n s'exécute à heure fixe (ou sur événement). Il récupère un sujet via Google News RSS ou une liste de sujets prédéfinis
  2. Génération de contenu — le workflow appelle l'API Claude avec un prompt structuré. Le résultat est un fichier HTML complet, formaté selon le template du site
  3. Push GitHub — n8n utilise l'API GitHub pour créer ou mettre à jour le fichier dans le dépôt du site. Un commit est créé automatiquement
  4. Déploiement Vercel — Vercel détecte le push via webhook et redéploie le site statique. L'article est en ligne en moins de 90 secondes après le commit

Ce dont vous avez besoin pour reproduire ça

  • n8n auto-hébergé sur un VPS (ex. Hostinger KVM 2, ~7 €/mois) — la version cloud n8n.io fonctionne aussi [1]
  • Un dépôt GitHub avec votre site statique (HTML, pas de Next.js ni WordPress requis)
  • Vercel connecté à ce dépôt — le plan gratuit suffit pour un site statique [2]
  • Une clé API Claude (Anthropic) pour la génération de contenu

Les limites honnêtes

Ce pipeline génère des articles de veille — pas du contenu de fond. Les articles produits automatiquement ont besoin d'être relus, sourcés, et corrigés avant d'être indexés par Google. Le présent article était initialement en noindex pour cette raison.

Ce type d'automatisation est utile pour publier régulièrement du contenu de valeur, mais ne remplace pas une stratégie éditoriale. C'est un outil de productivité, pas un rédacteur autonome.
